use strict là một chỉ thị được giới thiệu trong ECMAScript 5 (ES5) để cho phép một chế độ nghiêm ngặt hơn khi viết mã JavaScript. Khi bạn sử dụng use strict, JavaScript sẽ thực thi một tập hợp các quy tắc và hạn chế chặt chẽ hơn, giúp bạn tránh được một số lỗi phổ biến và các hành vi không mong muốn.

  • Trong chế độ không nghiêm ngặt, bạn có thể vô tình khai báo một biến mà không sử dụng từ khóa var, let, hoặc const, và biến này sẽ trở thành biến toàn cục. Trong chế độ nghiêm ngặt, điều này sẽ gây ra lỗi. 

                        "use strict";
                    x = 3.14; // Gây ra lỗi, vì x không được khai báo

  • Ngăn chặn việc gán giá trị cho các thuộc tính không thể ghi (read-only properties) 

                        "use strict";
                    Object.defineProperty(this, "x", { value: 42, writable: false });
                    this.x = 9; // Gây ra lỗi, vì x là read-only

  • Ngăn chặn khai báo các biến trùng lặp trong một hàm

                        "use strict";
                    function sum(a, a, c) { // Gây ra lỗi, vì tên biến a bị trùng
                        return a + a + c; 
                    }

  • Cấm việc sử dụng từ khóa with vì nó làm cho mã khó đọc và bảo trì. 

                        "use strict";
                    with (Math) { // Gây ra lỗi
                        x = cos(2);
                    }

  • Ngăn chặn việc sử dụng this là undefined trong các hàm, Trong chế độ nghiêm ngặt, nếu bạn gọi một hàm mà không có đối tượng cụ thể (this), this sẽ là undefined thay vì là đối tượng toàn cục (window trong trình duyệt) 

                        "use strict";
                    function f() {
                        return this; // Trong chế độ nghiêm ngặt, `this` sẽ là `undefined`
                    }
                    console.log(f()); // undefined

  • Bắt buộc khai báo trong các hàm : Chế độ nghiêm ngặt yêu cầu tất cả các biến phải được khai báo trước khi sử dụng. 

                        "use strict";
                    function test() {
                        undeclaredVar = 5; // Gây ra lỗi, vì biến chưa được khai báo
                    }

  • Ngăn chặn việc khai báo lại các từ khóa đã được bảo lưu

    Chế độ nghiêm ngặt ngăn chặn việc sử dụng một số từ khóa nhất định như implements, interface, let, package, private, protected, public, static, yield làm tên biến hoặc hàm.
